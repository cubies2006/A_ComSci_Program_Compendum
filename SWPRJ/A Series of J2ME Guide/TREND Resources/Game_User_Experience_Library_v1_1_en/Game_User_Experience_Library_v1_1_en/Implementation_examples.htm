Implementation examples

The table below contains examples of how to implement usability in mobile games. These games were evaluated as a part of creating the Series 40 Developer Platform 1.0: Usability Guidelines for J2ME™ Games document available at www.forum.nokia.com.

Lack of feedback

When throwing the ball, there is not enough feedback.

The user does not know that the character can be moved when the ball has been thrown.

In-game help is provided. The user sees that the character can move and which buttons to press to do this. Both number and arrow keys work; the symbols are arrows and they are marked with numbers.

The player does not necessarily realize that points have been scored and how many points are earned. There is not enough feedback.

Clear feedback is provided. The player learns the difference between different throws and the scored points.

If the player misses two throws in a row, the next round begins. There are four rounds altogether. The only feedback for ending a round is that the number changes at the top of the screen.

Clear feedback is provided to indicate that the round has changed. This also brings the concept of game rounds to the user’s attention.

While waiting for the user to do something, the character on the screen does nothing and is completely frozen.

To create some activity and make the character look alive, the character begins to bounce the ball while waiting for the user to do something.

Main menu

The main menu is purely text-based and does not differ from the UI in the rest of the device. It doesn't feel like a game and not many commands fit on the screen at one time.

The menu has been replaced by a graphic version with the theme of the game. Commands fit on the screen nicely and the whole appearance is more pleasing.

New game mode

There is only one game mode, which becomes repetitive after a while.

The game concept has been expanded to include several game modes. On the left is a picture of a multiplayer version (hot seat).

Here is another new game mode, which is time-limited. The player must score as many points as possible in a limited time.

Game world to match real world

The game has an invisible barrier to the right of the spaceship — the player cannot go right, although there is nothing visible preventing him.

The new version does not contain such barriers. Note also the little information circle where a player can get hints on how to play the game.

Secret levels

The secret levels are accessed after the current level has been completed and only if the player has collected a certain item to open the bonus level.

The bonus level is accessed directly after reaching the item, which makes the connection more obvious to the user.

In-game help

Players have the wrong impression about game characters and their roles.

A window appears the first time this creature is encountered. It tells the user important details about the creature.

Game menu

The game has two different menus, one depicted on the left. The menu does not look like it is part of a game, and scrolling is almost always needed.

The new version has only one menu, similar to the one on the left. (Note: The contents of the menu are different from the one shown here; pay attention only to the visual appearance). This menu is clearly different from the device UI in general.
