Main menu implementation

    • The game menu should be implemented with custom graphics, but only if it can be done well.

    • Make the menus look like they are part of the game, instead of part of the device. The application should be recognizable by the menu background colors, fonts, and so on. However, when implementing this, conserve device memory. Use the same graphics as in the game itself, or use only coloring instead of different graphics. Implement this only if it can be done well — it should work at least as well as the higher-level UI. Also, using image collections instead of separate images will help conserve heap memory usage.

 

    • Especially Java™ ME developers should pay attention in implementing the games main menu. In S60 devices, using full-screen mode and custom graphics is recommended.

    • In Java™ ME applications, using standard UI components in the game itself should be avoided.
