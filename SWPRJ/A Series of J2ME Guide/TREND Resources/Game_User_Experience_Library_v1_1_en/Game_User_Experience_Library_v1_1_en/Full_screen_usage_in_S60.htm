Full-screen usage in S60 

Mobile applications use the full-screen mode for different reasons: For example, pictures and videos are typically best viewed without titles or softkey labels. In addition to that, many games and entertainment applications have a custom user interface, and use full screen for the good looks. This article concentrates on the usability issues when using the full-screen mode in S60 applications, but many issues are relevant also to other platforms.

The basic principles are simple:

1.       Use the full-screen mode when it’s appropriate.

2.       It is OK for the UI to look different.

3.       Looks may vary, but the application behavior should remain the same.

Keep these principles in mind, and your application will be viewed positively not only by the application certification testers, but most importantly: the users!

Be conscious of your motives to use full screen

First of all, in order for you to keep the interfaces as user-friendly as possible, you need to know what you want to achieve by using the full screen mode. This simple self-test should help you to clarify your thoughts:

Sum up the answers in the A, B, and C columns, and select an appropriate interpretation from the following alternatives.

If you got mostly A answers:

So, you want to show everything at once? Pay special attention to these issues:

    • Remember that titles and softkey labels are important information for the user: Consider displaying them as a part of the full-screen mode, but use less screen space and a slightly smaller font.

    • Keep the navigation efficient. You don't need to show everything at once, if navigating is fast enough. Instead of forcing the user to scroll 20 pages of text, allow him to navigate easily between different subtitles.

    • Placing Full screen in the Options menu is not the only solution. Consider switching automatically to full screen when necessary, and back to the normal mode when you don't need to display that much information.

    • Instead of having a Full screen item in the Options menu, prefer having the full-screen mode as an option in the application’s Settings screen.

If you got mostly B answers:

So, you want to go for the good looks? Keep in mind that the behavior of the application is also important! You need to pay special attention to these issues:

    • You want to be perceived as interesting, not strange. Always use softkeys consistently.

    • S-L-O-W navigation is guaranteed to frustrate users, even with those fancy 3D animations. Keep navigation reasonably fast.

    • Not showing softkey labels can make users feel insecure. In most cases, it is far better to display small softkey labels than not showing them at all. Make users feel safe at all times.

    • Combine good looks with familiarity. Softkeys should behave normally, even if no labels are displayed.

    • Fancy design shouldn’t get in the way of good usability. Maintain a good contrast between the softkey labels and the background.

    • Consider displaying softkey labels for a few seconds, and then fading them away.

Figure: The title and the softkey labels follow the game’s visual style

If you got mostly C answers:

Sorry, but full screen might not be the best option for you. You should think again if you really need the full screen mode at all. Pay special attention to these issues:

    • The full-screen mode can be used to overcome some softkey label problems in S60 Java™ ME applications.

    • Consider not having the full-screen mode at all, at least not as the default mode.

    • If everybody should be able to use the application, it is better to display softkey labels.

    • Use a font that is large enough and easy to read for all users.

Unique, but consistent

The mobile user interface can be consistent with the S60 UI style in two ways: It can follow the visual style and the functionality. If you choose to use the full-screen UI, you are free to experiment with different visual styles. Using the full-screen mode allows UI designers more artistic freedom. You can design icon-based user interfaces for your application, or try fancy animated 3D menus for a more innovative and unique look. Also the softkey labels can be designed to be part of the visual style.

Figure: The current place and the means with which the user can move forward are communicated by the title and the softkey labels

However, with freedom comes responsibility: It is easy to mess things up! But even if the UI is visually different from the S60 UI style, the basic interaction principles should still remain the same: For example, the basics of navigation. It is always important to tell users clearly where they are, what they can do, and where they can go next.

Figure: There are several possibilities to communicate to the user his or her current place and the means to move on

In the figure, the rightmost figure shows the playing view — that is why the softkey labels are not displayed.

 

Figure: Different amounts of space can be reserved for the title and the softkey labels

As you can see from the previous figure, there is no big difference between displaying small labels and not displaying labels at all.

Even if your softkey labels are not displayed, they should still be easy to reach and behave the way the user expects. The user's motoric memory is likely to be far more powerful than the visual look and feel of the screen. This means that navigating backwards should always be possible with the right softkey, even if the Back label is not shown.

Figure: In both views, the user has to be able to navigate backwards with the right softkey

Basically the only good excuse for hiding softkey labels completely is displaying images, videos, or game graphics in full screen. This solution can be suitable for applications that use full screen only in certain modes, and both softkeys are used for pausing or navigating backwards.

Keep the following interaction basics in mind:

1.       Navigation key: Moving focus, selecting items, default actions. Note that backwards navigation should not be the default action!

2.       Left softkey: Opens the Options menu, or works as a secondary key for selecting. Default actions are also available from the Options menu.

3.       Right softkey: Safe backwards navigation.

Using the full-screen mode in a user-friendly way requires some thought, but is quite simple at the end. There are just some key principles you have to take into account.

To sum up:

1)       Know why you use full screen.

2)       It is OK to for the application to look different.

3)       The application must always behave normally.
