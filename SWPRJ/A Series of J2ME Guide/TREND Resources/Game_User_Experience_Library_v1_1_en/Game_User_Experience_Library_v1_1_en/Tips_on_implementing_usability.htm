Tips on implementing usability

In order to make implementing usability easier, the following tips have been drafted for developers.

Test early and often

The ultimate test for a user interface is to test it with real users. Testing is not a single event, but an iterative process, and it is better to perform several small tests than a single large one, since this allows more iterations.

A little testing is better than none at all

Even if testing at the recommended level is impossible, it is better to do some testing than to abandon it completely. The first testing sessions are usually the most valuable ones.

Choose simplicity

Keep things as simple as possible. A complicated solution is itself a problem.

The user is always right

An unsuccessful user interface cannot be excused by saying that the user is stupid. While it may be true that another user is capable of handling the situation, developers don’t have the option of selecting their users. If they do, they give up a very significant source of revenue and sales. Developers are designing their games for the user, and regardless of the technical merit of a solution, if nobody uses it, it is worthless.

Do not guide the user

When testing with users, developers should let them act as they normally do and not tell them where the options are or what key to press. Developers must be neutral and in the background. They should observe the users, not influence them.
