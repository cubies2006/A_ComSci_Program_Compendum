Help

Concentrate on essential issues in the help

Even though many players claim they read the help text, most do not — until they encounter a problem. Even when they do read the text, they typically remember only what they are looking for — most often, the controls. Some players also want to know what the goal of the game is.

    • The help text should be brief.

    • Provide help on the controls first, because this is the first thing players look for.

    • The game objective should be presented second, right after the controls. The objective should be described in four sentences or less.

    • The game story can be skipped altogether or mentioned only briefly. If the story is mentioned, it should be described after the controls, with the game goal.

    • Mention diagonal movement, if it is available (1, 3, 7, and 9).

Use pictures in the help

Help text typically describes the game controls and possibly the story. An illustration representing the effects of different keys is a better way to explain game play than plain text; arrows are easier to understand than plain text.

    • Use pictures to explain the controls, if possible. This works particularly well for movement controls. However, the pictures should not show the keypad and its buttons, but a character from the game and arrows representing its movement.

Help for multiplayer games

There can be separate instructions about multiplayer game features. Help for establishing multiplayer games should be placed in the screen where the game is established. Here, players can get instructions for creating connections and special functionalities in multiplayer games.

    • Provide help for establishing multiplayer games in the same menu where the multiplayer game is started.

    • Provide instructions if the multiplayer game works with a different logic than the single-player game.

          

    • On platforms with multitasking capability, tell users how the game is paused when players are interrupted. This is essential because players may not be aware of their device’s multitasking ability and therefore a paused game may be lost.

Due to technical constraints in some devices, the user might face problems with background music, tones that do not follow the device profile, and backlight that does not stay on long enough. In those devices:

    • Tell in the help or in the Sound menu that if the user cannot hear background music, even if it was turned on in the game settings, the device profile that is set to silent needs to be changed.

    • Tell in the help which button to push if the backlight goes off during the game.
